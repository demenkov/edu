ТЕЗИСЫ ДОКЛАДА
Тема: «Методы оптимизации хранения и обработки данных при синхронизации
разнородных источников с аналитическим хранилищем»
Деменков И.М., группа 12-25 РПмз. Научный руководитель — проф. Гришин И.Ю.

----------------------------------------------------------------------
1. Актуальность
- Рост объёмов данных и роль аналитики повышают требования к наполнению
  облачных колоночных хранилищ: стоимость, скорость, согласованность,
  доступность.
- Интеграция разнородных источников (MySQL, MongoDB) с единым хранилищем —
  нетривиальная задача.
- Существующие подходы к инкрементальной синхронизации не дают
  одновременно непрерывную доступность, полноту и приемлемую стоимость.

2. Объект и предмет
- Объект: процессы хранения и обработки данных при синхронизации
  разнородных источников с аналитическим хранилищем.
- Предмет: модели, методы и средства их оптимизации для повышения
  производительности аналитических запросов.

3. Цель и задачи
- Цель: аналитический обзор, обоснование подхода и разработка
  концептуальной и логической моделей оптимизированной синхронизации.
- Задачи: обзор и сравнение стратегий → постановка задачи → обоснование
  подхода → требования → модели (UML/ER) → анализ свойств и преимуществ.

4. Ключевые проблемы (мотивация)
- Окно недоступности: при DELETE + INSERT данные временно отсутствуют в
  таблице, запросы возвращают неполный результат.
- Пропуск строк: при инкременте по ID незафиксированная строка с меньшим
  ID «перешагивается» курсором по максимальному ID.
- Согласованность чтений: неполные промежуточные состояния таблицы.

5. Предлагаемый подход (процесс синхронизации)
- Промежуточная таблица как клон целевой → чтение «хвоста» с перекрытием
  (≈2 ч) → загрузка инкремента → обновление целевой таблицы.
- Штатно — атомарная замена «хвоста» (нет окна недоступности).
- Повторная синхронизация — слияние по ключу без удаления (идемпотентно).
- Перекрытие трейлингового окна устраняет пропуск строк.

6. Методы оптимизации хранения и запросов
- Партиционирование по дню (created_at) и кластеризация по домену клиента
  снижают объём сканирования и стоимость запросов.
- Доступ клиентов — через материализованные представления (MV) к общему
  датасету вместо копирования по датасетам: одна копия вместо n + 1,
  быстрее загрузка, согласованный доступ.

7. Модели и требования
- Концептуальная модель — UML-диаграмма деятельности цикла синхронизации.
- Логическая модель — ER: источник → промежуточная таблица (клон) →
  целевая таблица; ключ слияния, партиционирование, кластеризация.
- Требования: функциональные и нефункциональные (доступность, полнота,
  идемпотентность, согласованность, масштабируемость, наблюдаемость).

8. Выводы
- Проведён обзор и поставлена задача с выявленными ограничениями.
- Обоснован подход, сформулированы требования.
- Разработаны и формализованы средствами UML/ER концептуальная и
  логическая модели.
- Выполнен анализ свойств, ограничений и преимуществ перед аналогами.
